・リポジトリ
Git が管理するプロジェクトの全てのファイルと履歴を含むデータベース。
リポジトリには、ファイルの変更履歴、コミット、ブランチ、タグ、設定などが含まれる。

・リモートリポジトリ
リポジトリのうち、ネットワーク上の別のコンピュータにあるリポジトリ。
複数人で開発を行う場合、共有のリモートリポジトリにコミットをプッシュして共有することができる。

・ローカルリポジトリ
リポジトリのうち、自分のコンピュータにあるリポジトリ。

・ブランチ
リポジトリ内の、ある時点でのコードベースから分岐して独立して開発を進めることができる「分岐」。
新しい機能の開発やバグ修正を行う場合などに使用される。

・リモートブランチ
リモートリポジトリ上にある、リモートリポジトリのブランチ。

・ローカルブランチ
ローカルリポジトリ上にある、ローカルリポジトリのブランチ。
開発者が新しい機能やバグ修正を行う場合に、ローカルブランチを作成し、そこで変更を加えることができる。

・デフォルトブランチ
リポジトリの新しいブランチが作成されたときに自動的にチェックアウトされるブランチ。
通常は「main」または「master」と呼ばれるブランチがデフォルトブランチに設定されている。

・ワークツリー
リポジトリの中で、実際にファイルの編集や変更を行う場所。
ワークツリーは、ローカルリポジトリのファイルを直接変更する場所。

・インデックス / ステージ
Git において、変更をコミットする前にコミットに含める変更を選択するための仕組みです。
ワークツリーで変更されたファイルは、まずインデックスに追加され、次にコミットされる対象としてステージングされる。
インデックスに含まれる変更をコミットすることで、ローカルリポジトリに変更履歴が記録されます。
